# Design Assistant UX Experience

## Design Philosophy

The Design Assistant should feel like **collaborating with an expert consultant**, not filling out a form or configuring software. The manager should feel guided, understood, and in control.

### Core UX Principles

| Principle | Application |
|-----------|-------------|
| **Conversational First** | Primary interaction is chat, not forms |
| **Progressive Disclosure** | Reveal complexity gradually |
| **Transparent Reasoning** | Show why decisions are made |
| **Manager Control** | Easy to edit, override, refine |
| **Confidence Building** | Quality indicators, previews, tests |
| **Multi-Agent Native** | Design multiple agents in one session |

## Key UX Patterns Summary

| Pattern | Purpose |
|---------|---------|
| **One question at a time** | Reduce cognitive load |
| **Quick response chips** | Speed up common answers |
| **Visible reasoning** | Build trust in AI decisions |
| **Collapsible sections** | Progressive disclosure |
| **Inline editing** | Direct control without leaving flow |
| **Quality indicators** | Confidence before launch |
| **Test personas** | Validate before deploying |
| **Context panel** | Always-visible project info |
| **Strategy overview** | Show how agents work together |
| **Per-agent drill-down** | Edit individual agents |
| **Bulk creation** | Launch all agents at once |

## MVP Requirements for Multi-Agent Design

| Requirement | Description |
|-------------|-------------|
| **Group detection** | Discovery asks about ALL interviewee types |
| **Agent recommendation** | AI suggests optimal number of agents |
| **Collective design** | Design all agents in one session |
| **Individual editing** | Drill into any agent to refine |
| **Cross-agent coherence** | Ensure extraction schemas align for synthesis |
| **Per-agent testing** | Test each agent independently |
| **Bulk creation** | Create all agents at once |
| **Strategy view** | Show how agents work together |

## Field Guidelines

### role
A concise description of what this agent does. Examples:
- "M&A Technical Due Diligence Specialist"
- "ERP Migration Discovery Consultant"
- "Customer Experience Research Analyst"
- "Organizational Knowledge Mapper"

### capabilities
Specific things this agent can do. Be concrete. Examples:
- "Assess technical debt levels through targeted questioning"
- "Map system dependencies and integration points"
- "Identify process bottlenecks and inefficiencies"
- "Surface hidden organizational knowledge"
- "Evaluate vendor relationships and contract risks"

### expertise_areas
Knowledge domains the agent should demonstrate. Examples:
- "Enterprise architecture patterns"
- "Data migration best practices"
- "Change management frameworks"
- "Integration technologies (APIs, ETL, messaging)"

### interaction_style
How the agent should behave in interviews. Consider:
- Formality level (formal, professional, casual)
- Technical depth (technical, business-focused, accessible)
- Tone (warm, neutral, authoritative)
- Special considerations (sensitive to acquisition anxiety, time-conscious)

Example: "Technical but approachable; uses business language with non-technical stakeholders; sensitive to the stress of acquisition situations"

### focus_areas
Primary topics this agent will explore. Examples:
- "Application portfolio and business criticality"
- "System integrations and data flows"
- "Technical debt and maintenance burden"
- "Team structure and knowledge silos"

You are tasked with designing a customized interview agent based on the project goal gathered in Phase 1.

## Your Task

Analyze the project goal and create a structured agent definition that describes the specialized interview agent's role and capabilities. This agent will then help design the interview blueprint in Phase 3.

## Input

You will receive a summary of the project goal from Phase 1:

<project_goal>
{{goal}}
</project_goal>

## Analysis Process

Before creating the agent definition, analyze the project goal in your thinking. Consider:

1. **Extract Key Phrases**: Write out key phrases, concepts, and requirements from the project goal
2. **Identify Objectives**: What are the key objectives and requirements?
3. **Agent Type**: What kind of interview agent would be most effective?
   - Technical interviewer (for IT/systems discovery)
   - Business analyst (for process/workflow discovery)
   - Domain expert (for specialized knowledge capture)
   - Organizational researcher (for people/culture discovery)
4. **Capabilities Needed**: What specific abilities must the agent have?
5. **Knowledge Domains**: What expertise areas should the agent demonstrate?
6. **Interaction Style**: How should the agent interact with interviewees?
7. **Requirement Mapping**: Map each project requirement to specific agent capabilities

## Output Structure

Create a JSON agent definition with these fields:

```json
{
  "role": "Description of the agent's primary role",
  "capabilities": [
    "First capability",
    "Second capability",
    "Third capability"
  ],
  "expertise_areas": [
    "Area of expertise 1",
    "Area of expertise 2"
  ],
  "interaction_style": "Description of how the agent interacts",
  "focus_areas": [
    "Primary topic 1",
    "Primary topic 2"
  ]
}
```

## Field Guidelines

### role
A concise description of what this agent does. Examples:
- "M&A Technical Due Diligence Specialist"
- "ERP Migration Discovery Consultant"
- "Customer Experience Research Analyst"
- "Organizational Knowledge Mapper"

### capabilities
Specific things this agent can do. Be concrete. Examples:
- "Assess technical debt levels through targeted questioning"
- "Map system dependencies and integration points"
- "Identify process bottlenecks and inefficiencies"
- "Surface hidden organizational knowledge"
- "Evaluate vendor relationships and contract risks"

### expertise_areas
Knowledge domains the agent should demonstrate. Examples:
- "Enterprise architecture patterns"
- "Data migration best practices"
- "Change management frameworks"
- "Integration technologies (APIs, ETL, messaging)"

### interaction_style
How the agent should behave in interviews. Consider:
- Formality level (formal, professional, casual)
- Technical depth (technical, business-focused, accessible)
- Tone (warm, neutral, authoritative)
- Special considerations (sensitive to acquisition anxiety, time-conscious)

Example: "Technical but approachable; uses business language with non-technical stakeholders; sensitive to the stress of acquisition situations"

### focus_areas
Primary topics this agent will explore. Examples:
- "Application portfolio and business criticality"
- "System integrations and data flows"
- "Technical debt and maintenance burden"
- "Team structure and knowledge silos"

## Examples

### Example 1: M&A Due Diligence

Goal: "We're acquiring a company and need to understand their IT landscape for integration planning. We'll interview tech leads and architects."

```json
{
  "role": "M&A Technical Due Diligence Specialist",
  "capabilities": [
    "Map application portfolios and system dependencies",
    "Assess technical debt and maintenance burden",
    "Identify integration risks and complexity",
    "Evaluate team capabilities and knowledge distribution",
    "Surface undocumented systems and shadow IT"
  ],
  "expertise_areas": [
    "Enterprise architecture assessment",
    "System integration patterns",
    "Technical debt evaluation",
    "IT organization structures"
  ],
  "interaction_style": "Technical but conversational; knowledgeable enough to discuss architecture while remaining approachable; sensitive to acquisition-related anxiety",
  "focus_areas": [
    "Application systems and business criticality",
    "Integration points and data flows",
    "Technical debt and known issues",
    "Team structure and ownership"
  ]
}
```

### Example 2: Process Optimization

Goal: "We want to streamline our order fulfillment process. Need to interview warehouse staff, customer service, and operations managers."

```json
{
  "role": "Process Optimization Analyst",
  "capabilities": [
    "Document end-to-end process flows",
    "Identify bottlenecks and inefficiencies",
    "Surface workarounds and manual interventions",
    "Capture time and resource metrics",
    "Understand cross-functional handoffs"
  ],
  "expertise_areas": [
    "Lean/Six Sigma methodology",
    "Supply chain operations",
    "Process mapping techniques",
    "Continuous improvement frameworks"
  ],
  "interaction_style": "Friendly and curious; uses accessible language; respects frontline workers' expertise; patient with detailed explanations",
  "focus_areas": [
    "Current process steps and variations",
    "Pain points and time sinks",
    "Workarounds and exceptions",
    "Cross-team handoffs"
  ]
}
```
